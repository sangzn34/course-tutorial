## บทที่ 1 — เก็บ state ตะกร้าด้วย Jotai (global state)

### concept

ตะกร้าต้อง "จำ" ของข้ามหน้า — เพิ่มจากหน้า menu, ดูที่ drawer, ใช้ตอน checkout. ถ้าเก็บด้วย `useState` ในหน้าเดียว พอเปลี่ยนหน้าก็หาย. ต้องใช้ **global state** — state ที่ทุก component เข้าถึงได้

โปรเจคนี้ใช้ **Jotai** — library ที่มองทุกชิ้น state เป็น "atom" (หน่วยเล็ก ๆ) แล้ว component ไหนก็ subscribe atom ที่ต้องการได้

### โค้ด — [lib/cart.ts](../../lib/cart.ts)

```ts
export const cartAtom = atomWithStorage<CartItem[]>("cart", []);
```
- `atomWithStorage` = atom ที่ **sync กับ localStorage อัตโนมัติ** → ตะกร้ารอด refresh / ปิดเปิด browser (key `"cart"`, ค่าเริ่มต้น `[]`)

มี atom 3 แบบ แต่ละแบบใช้ hook ต่างกัน:

| Atom | ทำอะไร | hook ที่ใช้ |
|---|---|---|
| `cartAtom` | เก็บรายการในตะกร้า (อ่าน+เขียน) | `useAtom` (ได้ทั้งค่า + ฟังก์ชันเซ็ต) |
| `cartCountAtom` | นับจำนวนรวม — **คำนวณจาก `cartAtom`** | `useAtomValue` (อ่านอย่างเดียว) |
| `addToCartAtom` | สั่งเพิ่มของ — **เขียนอย่างเดียว** | `useSetAtom` (ได้แค่ฟังก์ชัน) |

`cartCountAtom` เป็น **derived atom** (atom ที่คำนวณจาก atom อื่น) — พอ `cartAtom` เปลี่ยน ตัวเลขนับก็อัปเดตเอง:
```ts
export const cartCountAtom = atom((get) => {
  const cart = get(cartAtom);
  return cart.reduce((total, item) => total + item.quantity, 0);
});
```

`addToCartAtom` เป็น **write-only atom** — ตรรกะ "ถ้ามีของชิ้นนี้แล้วเพิ่มจำนวน ไม่งั้นเพิ่มชิ้นใหม่" รวมไว้ที่นี่ที่เดียว ไม่ต้องเขียนซ้ำในทุก component:
```ts
export const addCart = (items, item) => {
  const existing = items.find((i) => i.productId === item.productId);
  return existing
    ? items.map((i) => i.productId === item.productId
        ? { ...i, quantity: i.quantity + item.quantity } : i)
    : [...items, item];
};
```

### ⭐ takeaway

- state ที่ต้องใช้ข้ามหน้า/หลาย component → global state (Jotai atom) ไม่ใช่ `useState`
- `atomWithStorage` = ได้ persistence ฟรี (localStorage)
- เลือก hook ตามการใช้: อ่าน+เขียน `useAtom` / อ่าน `useAtomValue` / เขียน `useSetAtom`
- รวม logic การแก้ state ไว้ใน atom (เช่น `addToCartAtom`) ดีกว่ากระจายในหลาย component

### ⚠️ จุดต้องแก้ที่เจอ
มี import ขยะ `import { init } from "next/dist/compiled/webpack/webpack"` ใน [lib/cart.ts](../../lib/cart.ts) — น่าจะมาจาก autocomplete พลาด, ลบทิ้ง (ไม่ได้ใช้)

## บทที่ 2 — ปุ่มตะกร้าลอย + Drawer

### concept

UI ตะกร้ามี 2 ส่วน: **ปุ่มลอย** (มุมจอ โชว์จำนวน) และ **drawer** (แผงเลื่อนเข้ามาจากขวา แสดงรายการ)

### ปุ่มลอย — [components/CartButton.tsx](../../components/CartButton.tsx)
- ลอยมุมขวาล่าง: `fixed bottom-6 right-6 z-40` (`fixed` = ติดหน้าจอไม่เลื่อนตาม, `z-40` = ลอยทับเนื้อหา)
- โชว์ badge แดงเฉพาะตอน `count > 0`
- ปุ่มนี้คุม state `open` ของ drawer เอง (`useState`)

### Drawer — [components/CartDrawer.tsx](../../components/CartDrawer.tsx)
แผงเลื่อนจากขวา โครงสร้าง:
```
[ฉากหลังดำโปร่ง]  ← คลิกเพื่อปิด
[แผง drawer]
  ├── รายการสินค้า (ชื่อ + ราคา + ปุ่ม [-] จำนวน [+] + ปุ่มลบ)
  ├── ยอดรวม
  └── ปุ่มไป checkout
```

logic หลัก (ล้วนเป็นการแปลง array แบบ immutable — สร้าง array ใหม่ ไม่แก้ของเดิม):
```ts
ลบ:     carts.filter(c => c.productId !== id)
จำนวน:  carts.map(c => c.productId === id ? {...c, quantity: c.quantity + delta} : c)
ยอดรวม: carts.reduce((sum, c) => sum + c.price * c.quantity, 0)
```
animation เลื่อนเข้า/ออก: สลับ class `translate-x-full` ↔ `translate-x-0` + `transition-transform duration-300`

### ⭐ takeaway

- แก้ state ใน React **ห้ามแก้ array/object เดิมตรง ๆ** — ใช้ `map`/`filter`/spread `{...}` สร้างใหม่เสมอ (ไม่งั้น React ไม่รู้ว่าเปลี่ยน เลยไม่ re-render)
- `fixed` + `z-index` = ทำของลอยทับหน้า (ปุ่ม, drawer, modal)

### 🟡 ข้อสังเกต
`CartDrawer` ใช้ jotai hooks แต่ **ไม่มี `"use client"`** — มันรอดเพราะ parent (`CartButton`) เป็น client อยู่แล้ว แต่ใส่ `"use client"` ให้ชัดจะดีกว่า (กันพังถ้าวันหลังมีคนเอาไปใช้ที่อื่น)

## บทที่ 4 — หน้า Checkout (form + ส่งข้อมูลขึ้น API)

### concept

หน้า [app/checkout/page.tsx](../../app/checkout/page.tsx) รวม 3 library:
- `react-hook-form` — จัดการฟอร์ม (เก็บค่า input, validate)
- `@tanstack/react-query` `useMutation` — ส่งข้อมูล**ขึ้น** server (POST/PUT/DELETE). คู่กับ `useQuery` ที่ใช้**ดึง**ข้อมูล
- `sonner` — toast แจ้ง success/error

```tsx
mutationFn: (data) => fetch("/api/order", {
  method: "POST",
  body: JSON.stringify({ ...data, items: carts }),   // รวมข้อมูลฟอร์ม + ของในตะกร้า
}).then(async (res) => {
  const body = await res.json().catch(() => null);
  if (!res.ok) throw new Error(getErrorMessage(body) ?? "เกิดข้อผิดพลาด");
  return body;
})
```
แล้วครอบ `try/catch` รอบ `mutateAsync` → สำเร็จ toast เขียว, error toast แดง

### ⭐ takeaway
- ดึงข้อมูล = `useQuery`, ส่งข้อมูลขึ้น (เปลี่ยนแปลงข้อมูล) = `useMutation`
- เช็ค `res.ok` เสมอ — `fetch` **ไม่ throw เองตอน server ตอบ 400/500**, ต้องเช็คแล้ว throw เอง

### 🟡 จุดต้องแก้
หลัง checkout สำเร็จ **ยังไม่ได้ล้างตะกร้า** — ของเก่าจะค้างไปออเดอร์หน้า ต้องเพิ่ม `setCarts([])` + `router.push(...)` ไปหน้าสำเร็จ

## บทที่ 5 — ป้องกัน Order API ฝั่ง server (บทเรียนสำคัญเรื่องความปลอดภัย)

### concept

**กฎเหล็ก: อย่าเชื่อข้อมูลที่ส่งมาจาก browser** ผู้ใช้ (หรือคนไม่หวังดี) แก้ค่าอะไรก็ได้ก่อนส่งมา server ต้อง validate + คำนวณราคาเองเสมอ

[app/api/order/route.ts](../../app/api/order/route.ts) ถูกแก้ให้แข็งแรงขึ้น 3 จุด:

**1. Validate input ด้วย Zod**

เดิมรับ body แบบ cast type ตรง ๆ (`as { items?: ... }`) = หลอก TypeScript เฉย ๆ ของจริงอาจไม่ตรง. เปลี่ยนเป็น Zod ตรวจของจริงตอน runtime:
```ts
const result = orderSchema.safeParse(payload);
if (!result.success) return  // ตอบ 400 + ข้อความ error ภาษาไทย
```

| เช็คเอง (if ทีละ field) | Zod |
|---|---|
| เขียน if เยอะ กระจาย | ประกาศ schema ที่เดียว |
| TypeScript ยังไม่รู้ type ที่ปลอดภัย | `result.data` ได้ type ที่ถูกต้องอัตโนมัติ |

**2. หาสินค้าด้วย Map แทน find ใน loop**

เดิม `products.find(...)` ข้างใน loop = ช้า (ของ N ชิ้น ลูกค้า M ชิ้น → ทำ N×M รอบ). เปลี่ยนเป็น Map (หาแบบ key ตรง ๆ เร็วกว่ามาก):
```ts
const productById = new Map(products.map((p) => [p.id, p]));
// แล้วในลูป: productById.get(item.productId)
```

**3. คำนวณราคาจาก DB ไม่ใช่จาก client (กันโกง)**

จุดอันตรายเดิม: ถ้าหา product ไม่เจอ โค้ดเก่าตั้ง `price: 0` เงียบ ๆ → ลูกค้าอาจสั่งของฟรี. แก้เป็น: ดึงราคาจาก DB (`unitPrice: product.price`), ถ้ามีสินค้าที่ไม่มีในระบบ → ตอบ 400 ปฏิเสธทั้งออเดอร์
```ts
const orderItems = items.flatMap((item) => {
  const product = productById.get(item.productId);
  return product ? [{ ...; unitPrice: product.price }] : [];  // ไม่เจอ = ข้าม
});
if (orderItems.length !== items.length) return  // 400 "มีสินค้าบางรายการที่ไม่มีอยู่ในระบบ"
```

### ⭐ takeaway (จำให้ขึ้นใจ)

- **ราคา/ยอดเงิน คำนวณที่ server จาก DB เสมอ** — อย่าเอาราคาที่ client ส่งมาใช้ตรง ๆ (ลูกค้าแก้เป็น 0 ได้)
- validate ทุก input ที่เข้ามา API ด้วย schema (Zod) — ไม่ใช่ cast type
- `find` ใน loop = ช้า; แปลงเป็น `Map` ก่อนถ้าต้องหาบ่อย

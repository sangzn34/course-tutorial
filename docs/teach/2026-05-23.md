## 1. Cart state (Jotai)

ใหม่: [lib/cart.ts](../../lib/cart.ts)

```ts
export const cartAtom = atomWithStorage<CartItem[]>("cart", []);
```

- `atomWithStorage` → sync `localStorage` อัตโนมัติ (cart รอด refresh)
- `addToCartAtom` — write-only atom: ถ้า productId มีอยู่แล้ว → เพิ่ม `quantity`, ไม่งั้น push ใหม่
- `cartCountAtom` — derived (read-only): รวม `quantity` ทุก item

### Pattern ที่สอน

| Atom | ใช้ใน component ด้วย |
|---|---|
| `cartAtom` (read+write) | `useAtom` — ทั้งอ่านและเขียน |
| `cartCountAtom` (derived) | `useAtomValue` — อ่านอย่างเดียว |
| `addToCartAtom` (write-only) | `useSetAtom` — เขียนอย่างเดียว |

## 3. Cart Drawer

[components/CartDrawer.tsx](../../components/CartDrawer.tsx) — slide-in panel จากขวา

### โครงสร้าง

```
[backdrop overlay (z-50)] ← click ปิด
[aside drawer]
  ├── items list
  │    ├── name + price
  │    ├── [-] qty [+]
  │    └── delete button
  ├── total
  └── checkout link
```

### Logic key

- **Delete**: `carts.filter(c => c.productId !== id)`
- **Quantity**: `carts.map(c => c.id === id ? {...c, quantity: c.quantity + delta} : c)` (กัน `< 1`)
- **Total**: `reduce((sum, c) => sum + c.price * c.quantity, 0)`
- **Animation**: `translate-x-full` ↔ `translate-x-0` + `transition-transform duration-300`

> 🟡 `CartDrawer` ไม่มี `"use client"` แต่ใช้ jotai hooks — work ได้เพราะ parent (`CartButton`) เป็น client; explicit จะดีกว่า

## 7. Order API hardening

[app/api/order/route.ts](../../app/api/order/route.ts)

### Before
- รับ body แบบ raw cast type `as { items?: ItemInput[] }`
- `products.find(p => p.id === item.productId)` ใน loop (O(n²))
- ถ้าหา product ไม่เจอ → `price: 0` เงียบๆ (เสี่ยงโกง free order)

### After
- **Zod validate** input ทั้งก้อน → 400 + error message ไทย
- **Map lookup** `new Map(products.map(p => [p.id, p]))` → O(n)
- `flatMap` skip product ที่ไม่มี → เช็ค `orderItems.length !== items.length` → 400 "มีสินค้าบางรายการที่ไม่มีอยู่ในระบบ"
- เปลี่ยน field `price` → `unitPrice` (ตาม schema order_items)
- ใช้ type `Prisma.OrderItemUncheckedCreateWithoutOrderInput` แทน inline object

### Why zod ดีกว่า manual check

| Manual | Zod |
|---|---|
| if-check ทุก field | declarative schema |
| ไม่มี type narrowing | `result.data` typed อัตโนมัติ |
| error message hardcode กระจาย | รวมที่ schema |

02 Mental Model

DB เก็บ "ที่อยู่รูป" ไม่ใช่ตัวไฟล์

FILE ไฟล์รูปจริง ผู้ใช้เลือกจากเครื่อง — ตัวไฟล์ binary
→
STORAGE local / Supabase เก็บไฟล์ตัวจริง — สลับได้ด้วย env
→
DATABASE imageUrl: string เก็บแค่ "ที่อยู่" ของรูป

ไฟล์ตัวจริงอยู่ที่ storage  ·  DB เก็บแค่ URL เป็น string — แยกหน้าที่กันชัด

⭐ เลือก storage backend ได้ด้วย env — schema และฝั่งแอปไม่ต้องรู้ว่าไฟล์ไปอยู่ไหน

08 Local → Supabase · ตอน Deploy

local FS ไม่รอด serverless — สลับ env อย่างเดียว

STORAGE_DRIVER=local เหมาะกับตอน dev
✓ติดตั้งง่าย ไม่ต้องต่อ service
✗ไฟล์หายทุกครั้งที่ deploy / rebuild
✗หลาย instance ไม่เห็นไฟล์ของกัน
→
STORAGE_DRIVER=supabase เหมาะกับ production
✓ไฟล์คงอยู่ ไม่หายตอน deploy
✓ทุก instance เห็นไฟล์ชุดเดียวกัน
✓เสิร์ฟผ่าน CDN ได้เลย
⭐ เพราะทำ adapter ไว้ตั้งแต่แรก การย้ายขึ้น prod เหลือแค่เปลี่ยน env — โค้ดเท่าเดิม
